7. Imagine que você está participando da análise de requisitos de um sistema de
reservas de hotel.
Liste 8 requisitos funcionais e 5 não funcionais que seriam importantes para
esse sistema

    Requisitos funcionais:
        1. O sistema deve registrar novos hóspedes através de uma conta pessoal.
        2. O sistema deve registrar nome, e-mail, telefone e senha de novas contas.
        3. O sistema deve fornecer ao usuário as opções de reservas disponíveis no hotel. 
        4. O usuário pode escolher qualquer tipo de reserva disponível, assim como o dia inicial e
            seu tempo de estadia.
        5. O sistema deve mostrar ao usuário todas as informações disponíveis sobre a reserva 
            antes de finalizá-la.
        6. O sistema deve fornecer diversas formas de pagamento para o usuário.
        7. O sistema deve informar automaticamente o usuário caso haja alguma mudança na sua reserva.
        8. O usuário deve poder consultar as informações de suas reservas mesmo após sua estadia,
            tais como início, fim, data e local.

    Requisitos não funcionais:
        1. O sistema deve ser simples e fácil de usar.
        2. O sistema não deve manter nem fornecer a terceiros dados pessoais de seus usuários.
        3. O sistema deve ser capaz de operar corretamente, sem falhas, por longos períodos de tempo.
        4. O sistema pode ser modificado, atualizado e mantido ao longo do tempo. 
        5. O sistema deve permanecer acessível 24 horas por dia, 7 dias por semana, exceto em
            horários de manutenção.
